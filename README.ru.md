# APR — Arch Package Remover

Интерактивный TUI для безопасного удаления пакетов в Arch Linux (и других
дистрибутивах на pacman/ALPM), написан с использованием
[ratatui](https://ratatui.rs).

[English version of README](README.md)

## Возможности

- Просмотр, поиск и фильтрация установленных пакетов
- Отметка нескольких пакетов для удаления
- **Настоящий dry-run**: перед удалением формируется реальная транзакция ALPM
  (с учётом транзитивных зависимостей)
- Отображение "Required By" (обратных зависимостей) для каждого пакета
- Поиск пакетов-сирот (orphans) — зависимостей, которые больше никому не нужны
- Быстрая команда **"отметить все сироты"** одной клавишей
- Встроенный экран справки со всеми горячими клавишами (`?`)
- Сортировка по имени, размеру или с приоритетом пакетов-сирот
- Несколько режимов удаления: простой (`-R`), рекурсивный (`-Rs`), полный
  (`-Rns`) и принудительный (`-Rdd`)
- Предупреждение при попытке удалить пакет из встроенного списка критически
  важных системных пакетов
- Отображение прогресса и лога во время удаления
- Обновление списка пакетов без перезапуска программы

## Установка

### Готовый бинарник

Каждый тегированный релиз (`vX.Y.Z`) автоматически собирается через GitHub
Actions и публикуется на странице
[Releases](https://github.com/TeddyBear/arch-pkg-remover/releases) как
`apr-vX.Y.Z-x86_64-linux.tar.gz`. Скачайте, распакуйте и запустите:

```sh
tar -xzf apr-vX.Y.Z-x86_64-linux.tar.gz
sudo ./apr
```

### Сборка из исходников

Требуется Rust (stable) и заголовки `pacman`/`alpm`, которые уже есть в
стандартной установке Arch Linux.

```sh
git clone https://github.com/TeddyBear/arch-pkg-remover.git
cd arch-pkg-remover
cargo build --release
```

Готовый бинарник появится в `target/release/apr`.

## Использование

APR изменяет базу данных пакетов, поэтому требует прав root:

```sh
sudo ./target/release/apr
```

### Горячие клавиши

| Клавиша        | Действие                                  |
|----------------|--------------------------------------------|
| `↑`/`↓`, `j`/`k` | Перемещение по списку                    |
| `g` / `G`      | В начало / в конец списка                  |
| `Space`        | Отметить/снять отметку с пакета            |
| `a`            | Отметить/снять отметку со всех видимых     |
| `O`            | Отметить все сироты (быстрая очистка)      |
| `/`            | Поиск (по имени, описанию, версии)         |
| `Esc`          | Очистить поиск / фильтр сирот              |
| `o`            | Включить/выключить фильтр "только сироты"  |
| `s`            | Переключить режим сортировки (Имя / Размер / Сироты) |
| `1`–`4`, `Tab` | Выбрать / переключить режим удаления (`-R`, `-Rs`, `-Rns`, `-Rdd`) |
| `d`            | Запустить dry-run и открыть диалог подтверждения |
| `r`            | Обновить список пакетов                    |
| `?`            | Показать экран справки                     |
| `q`            | Выход                                      |
| `Ctrl+C`       | Принудительный выход                       |

В диалоге подтверждения введите `yes` и нажмите `Enter`, чтобы выполнить
удаление, или `Esc` для отмены.

## Режимы удаления

| Режим      | Флаг   | Описание                                       |
|------------|--------|---------------------------------------------------|
| Simple     | `-R`   | Удалить только выбранные пакеты                    |
| Recursive  | `-Rs`  | Также удалить неиспользуемые зависимости           |
| Full       | `-Rns` | Recursive + удаление конфигурационных файлов       |
| Force      | `-Rdd` | Игнорировать проверки зависимостей (осторожно!)    |

## Замечания по безопасности

- Перед фактическим удалением всегда выполняется dry-run, который показывает
  полный список затрагиваемых пакетов (включая транзитивные зависимости,
  подтягиваемые рекурсивными режимами) и итоговый объём освобождаемого места.
- Если в список удаления попадает пакет, считающийся частью базовой системы
  (например, `glibc`, `systemd`, ядро `linux`, сам `pacman`), APR покажет
  предупреждение в диалоге подтверждения.
- Программа напрямую работает с базой данных pacman через libalpm.
  **Всегда внимательно читайте экран подтверждения перед вводом `yes`.**

## CI/CD

В репозитории настроен GitHub Actions (workflow-файлы в `.github/workflows/`):

- **`ci.yml`** — запускается при каждом push в `main` и в pull request'ах.
  Собирает проект в контейнере Arch Linux (debug + release), запускает
  `cargo fmt --check` и `cargo clippy`.
- **`release.yml`** — запускается при пуше тега вида `v*` (например,
  `v0.2.0`). Собирает release-бинарник, делает strip, упаковывает в
  `apr-vX.Y.Z-x86_64-linux.tar.gz` и публикует на странице
  [Releases](https://github.com/TeddyBear/arch-pkg-remover/releases) с
  автоматически сгенерированными заметками о релизе.

Чтобы выпустить новый релиз:

```sh
git tag v0.2.1
git push origin v0.2.1
```
